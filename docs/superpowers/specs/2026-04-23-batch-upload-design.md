# 批量上传功能设计

## 概述

在首页 UploadPage 支持批量上传 PDF 文件（最多 10 个），统一布局，一键并行提交到后端异步提取。

## 约束

- 最多 10 个 PDF 文件
- 仅支持电子版 PDF（无扫描件/OCR）
- 所有文件共用一个 PDF 布局设置（单栏/双栏）
- 后端零改动

## 状态改造

| 现状 | 改为 |
|------|------|
| `useState<File \| null>` | `useState<File[]>` |
| `<input>` 单文件 | 添加 `multiple` |
| 拖拽取 `files[0]` | 拖拽取全部 PDF，与已有文件合并 |

## 文件管理

- **去重**：按文件名去重，重复时 toast 提示
- **上限**：超过 10 个只取前 10，toast 提示
- **过滤**：非 PDF 文件直接跳过，toast 提示
- **删除**：每个文件行右侧有删除按钮

## UI 布局

```
┌─────────────────────────────────────────┐
│  📄 原始材料            已选择 N/10 个文件 │
├─────────────────────────────────────────┤
│  拖拽区（始终可见，支持追加拖拽）           │
│                                         │
│  📄 file1.pdf    2.3 MB          ✕      │
│  📄 file2.pdf    1.1 MB          ✕      │
│  📄 file3.pdf    4.7 MB          ✕      │
│                                         │
│  ─ ─ ─ 点击继续添加文件 ─ ─ ─            │
└─────────────────────────────────────────┘
```

- 无文件时：拖拽区显示完整上传引导
- 有文件时：拖拽区缩小为单行提示，下方展示文件列表
- 底部虚线按钮 "点击继续添加文件"

## 提交逻辑

- `Promise.all` 并行调用 `extractionService.startExtraction`
- 每个文件独立创建后端 task，独立轮询
- 提交期间按钮显示 "提交中... (N/M)" 进度
- 全部完成后跳转到 /records
- 部分失败：toast 提示具体文件名和原因，其余正常跳转
- 全部失败：停留在上传页

## 错误处理

| 场景 | 处理 |
|------|------|
| 非 PDF 文件 | toast + 跳过 |
| 超过 10 个 | 只取前 10 + toast |
| 同名文件 | 去重 + toast |
| 提交中删除 | 不允许，隐藏删除按钮 |
| 部分提交失败 | toast + 其余正常跳转 |
| 全部失败 | 停留上传页 |

## 改动范围

仅 1 个文件：`frontend/src/pages/UploadPage.tsx`

## 不改动的部分

- 后端 API：零改动
- documentsStore：无需改动
- extractionService：无需改动
- 路由/导航：无需改动
